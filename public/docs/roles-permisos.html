Roles y Permisos

Roles del sistema Ademy, accesos recomendados por módulo y codificación de permisos.

Roles del sistema

Usuario superadmin

El usuario con el rol de mayor jerarquía bypasea todos los checks de permisos en el middleware. No requiere tener los permisos explícitamente asignados en la tabla de roles.

Códigos de permisos

Los permisos usan notación modulo.accion. Se asignan a roles y se verifican en el backend con checkPermission('codigo').

Implementación técnica

En el backend, los permisos se verifican en el middleware de autenticación:

// Verificar un permiso específico
router.get('/api/pagos', verifyToken, checkPermission('pagos.listar'), getPagos);

// Verificar cualquiera de varios permisos (OR)
router.get('/api/reportes', verifyToken, checkPermission(['reportes.ver', 'admin.total']), getReportes);

Los menús del frontend también son dinámicos — se cargan desde la base de datos según los permisos del usuario autenticado via GET /api/usuarios/menu.
