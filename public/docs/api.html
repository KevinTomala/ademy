API Reference

Documentación de todos los endpoints del backend Ademy. Base URL: https://api.[tu-dominio]

Documento interno

Esta referencia está dirigida al equipo de desarrollo y administradores del sistema. Los códigos de permisos mostrados son ilustrativos del patrón modulo.accion y no representan un vector de ataque por sí solos — requieren credenciales válidas para ser utilizados.

Autenticación

Todos los endpoints protegidos requieren el header:

Authorization: Bearer <token>

Los tokens tienen una expiración de 4 horas. El frontend los almacena en localStorage y los invalida automáticamente ante una respuesta 401.

Códigos de respuesta

CódigoSignificado
200Consulta / actualización exitosa
201Creación exitosa
400Datos inválidos / falta de campos
401Token ausente, inválido o expirado
403Sin permiso para el recurso
404Recurso no encontrado
500Error interno del servidor

Ejemplo de uso

# 1. Login
curl -X POST https://api.[tu-dominio]/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"usuario@ejemplo.com","password":"********"}'

# 2. Llamada protegida
curl https://api.[tu-dominio]/api/pagos \
  -H "Authorization: Bearer <TOKEN>"

Nota sobre orden de rutas en Express

Rutas específicas deben declararse antes que rutas genéricas tipo /:id. En particular: GET /api/cursos/centro/:centro_id y GET /api/especialidades/nivel/:nivel_id pueden ser interceptadas por /:id si no están en el orden correcto.
