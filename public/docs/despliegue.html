Instalación local (Docker)

Prerrequisitos

  • Docker Desktop instalado y corriendo
  • Git
  • Node.js 20+ (para desarrollo sin Docker)

Pasos

Clonar el repositorio

Crear archivos .env

Copiar .env.example a .env en la raíz del monorepo y ajustar las variables.

Levantar los servicios

# Levantar MySQL + Backend + Frontend
docker compose up -d --build

# Incluir phpMyAdmin en :8080 (opcional)
docker compose --profile tools up -d

# Ver logs
docker compose logs -f backend
docker compose logs -f frontend

Variables de entorno

Backend (backend/.env)

# Base de datos
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=tu_password
DB_NAME=g_academico
DB_SSL=false

# JWT
JWT_SECRET=tu_clave_secreta_muy_larga

# CORS
FRONTEND_URL=http://localhost:3001

# Storage: "local" o "r2"
STORAGE_DRIVER=local
STORAGE_DIR=./storage

# Cloudflare R2 (solo si STORAGE_DRIVER=r2)
R2_ACCOUNT_ID=
R2_ACCESS_KEY_ID=
R2_SECRET_ACCESS_KEY=
R2_BUCKET_NAME=
R2_PUBLIC_URL=

# Email (Mailgun)
MAILGUN_API_KEY=
MAILGUN_DOMAIN=

# Modo de inicio: "blocking" | "nonblocking"
SERVER_START_MODE=blocking

Frontend (frontend/.env)

# URL del backend (todas las vars deben iniciar con REACT_APP_)
REACT_APP_API_URL=http://localhost:3000
Variables en Vercel

En Vercel las variables se configuran en Settings → Environment Variables. En producción: REACT_APP_API_URL=https://api.[tu-dominio]

DNS en Cloudflare para el dominio personalizado:

Tipo    Nombre                  Valor

Sin https:// en DNS

En Cloudflare DNS no se agrega el prefijo https:// en ningún campo. Solo el dominio desnudo.

Backend en Render

El backend usa SERVER_START_MODE=nonblocking en Render para que el servicio responda mientras espera la conexión a la base de datos (Render puede tardar en levantar).

Tip: Cold starts en Render

En el plan gratuito de Render el servicio se duerme tras 15 min de inactividad. El primer request puede tardar ~30s. Para producción real considera el plan Basic o superior.

Troubleshooting común

SíntomaCausa probableSolución
Frontend no conecta con backend REACT_APP_API_URL incorrecto Verificar la variable y reconstruir el build
Error CORS en producción FRONTEND_URL no coincide Actualizar FRONTEND_URL en Render con el dominio exacto
MySQL no inicia en Docker Puerto 3306 ocupado localmente Cambiar el puerto en docker-compose.yml o detener el MySQL local
Archivos no se suben Directorio de storage no existe Crear backend/storage/ o configurar R2
JWT inválido inmediatamente JWT_SECRET diferente entre deploys Usar el mismo secret en todos los entornos
